# Artic Store

Интернет-магазин одежды, обуви и аксессуаров — фронтенд-часть. Самостоятельно работающая сборка с единой дизайн-системой: главная, каталог, страница товара, подарочный сертификат, корзина, оформление заказа, авторизация, личный кабинет, информационные страницы и контакты — всё свёрстано и интерактивно.

**Демо:** https://romanzhan.github.io/artic-store/ — опубликованная сборка с демо-данными, весь функционал работает в браузере без установки.

## Технологии

- **Vite** — сборка и dev-сервер
- **SCSS** + нативные CSS-переменные — единая дизайн-система токенов (цвета, отступы, типографика, тени, z-уровни)
- **Vanilla JS** (ES-модули, делегирование событий) — без тяжёлых фреймворков
- **Handlebars** — переиспользуемые партиалы разметки
- **Swiper** — слайдеры и карусели
- **GLightbox** — полноэкранный просмотр фото товара
- **axios** — HTTP-клиент слоя авторизации (готов под Laravel Sanctum)
- **IMask** — маски ввода (телефон, email)
- Шрифты self-hosted (Inter, Noto Sans), изображения в WebP, иконки — единый SVG-спрайт

## Что реализовано

- **Главная** — hero-слайдер, карусели товаров, промо-секции, блог
- **Каталог** — фильтры (бренд / размер / цвет / цена), сортировка, пагинация и боковая навигация **без перезагрузки страницы** (SPA-навигация с человекопонятными URL `/catalog/{пол}/{группа}/{категория}`)
- **Страница товара** — галерея с миниатюрами и полноэкранным просмотром, переключение цвета и размера на месте, **таблица размеров** (модальное окно с подсветкой выбранного размера), табы «Описание / Доставка и возврат», блок «С этим товаром выбирают». Две покупки: «В корзину» и **«Купить сейчас»** (экспресс-оформление только этого товара, не затрагивая корзину)
- **Подарочный сертификат** — выбор суммы и вида (конверт / электронный), та же логика покупки, что и у товара
- **Корзина** — выезжающая панель: изменение количества, удаление, очистка с подтверждением, промокоды (применение/отмена и расчёт скидки), счётчик в шапке, сохранение между сессиями
- **Оформление заказа** — выезжающая панель: контактные данные с валидацией и масками (телефон, email), выбор доставки/самовывоза (адрес с автоподсказками или выбор филиала, дата и время), способ оплаты (наличный расчёт доступен только при самовывозе), единая сводка с промокодом, окно успешного оформления. Для авторизованных — переход к оформлению напрямую (без окна «Больше выгоды»)
- **Авторизация и регистрация** — набор модальных окон (вход, регистрация, подтверждение email с таймером, восстановление и смена пароля) с валидацией и масками; вход в контексте оформления возвращает к заказу
- **Личный кабинет** (`/account`, с защитой от неавторизованного доступа) — главная-дашборд с профилем и превью последних заказов и избранного; разделы с маршрутизацией по URL: профиль с редактированием данных (`/account/profile`), история заказов и страница деталей заказа (`/account/orders`, `/account/orders/{номер}`), избранное (`/account/favorites`). На мобильных вместо дашборда — меню-список разделов, у внутренних страниц кнопка «назад»
- **Информационные страницы** — Оплата (`/payment`), Доставка и возврат (`/delivery`), Политика конфиденциальности (`/privacy`), **Контакты** (`/contacts`) с формой обратной связи, реквизитами и **картой Яндекса** с филиалами и переключаемыми пинами
- **Страница 404** — оформленная страница для несуществующих адресов с навигацией; роутинг неизвестных путей работает в dev и на GitHub Pages
- **Избранное** — добавление из карточки, страницы товара и корзины, синхронизация во всех местах; список избранного — раздел личного кабинета, сердце в шапке ведёт туда
- **Адаптив** — десктоп и мобильные брейкпоинты 1440 / 1024 / 768 / 429, фиксированные мобильные CTA, корректная работа на «вырезах» iPhone (safe-area)

Данные временно отдаются из мок-слоя (`src/js/api/mockApi.js`) — структура подготовлена к подключению реального бэкенда; корзина и избранное сохраняются в браузере. Слой авторизации (`src/js/api/authApi.js`) построен на **axios** под Laravel Sanctum (cookie + CSRF) — методы входа/регистрации уже написаны под реальные эндпоинты, локально работает мок; подключение бэкенда — через `VITE_API_URL`.

Карта на странице «Контакты» подключается через **Яндекс.Карты**: ключ задаётся в `.env.local` как `VITE_YANDEX_MAPS_KEY` (шаблон — `.env.example`). Без ключа на месте карты показывается аккуратная заглушка, остальной функционал страницы работает.

## Как протестировать демо

Несколько подсказок к состояниям, до которых не всегда очевидно дойти (данные демонстрационные, хранятся в браузере):

- **Вход:** готового аккаунта нет — сначала пройдите регистрацию (иконка профиля в шапке → «Зарегистрироваться»), данные любые. **Код подтверждения из письма — любые 4–6 цифр** (например, `1234`). После входа из шапки открывается личный кабинет; выйти можно кнопкой там же.
- **Личный кабинет:** после входа доступны профиль, история заказов (демо-данные, у заказа есть страница деталей) и избранное; на мобильной ширине навигация устроена как меню-список.
- **Промокод:** в корзине или при оформлении введите `ARTIC15` (−15%) или `SALE` (−2500 ₽).
- **Сброс пароля:** в окне входа → «Забыли пароль?» → email → код (любые 4–6 цифр) → новый пароль.
- **Таблица размеров:** на странице товара ссылка «Таблица размеров» открывает окно; текущий выбранный размер подсвечен в таблице.
- **Купить сейчас:** на странице товара открывает оформление только с этим товаром — существующая корзина не меняется.
- **Страница 404:** откройте любой несуществующий адрес, например `…/artic-store/неизвестная-страница`.

## Запуск локально

```bash
npm install
npm run dev
```

Откройте `http://localhost:5173`.

## Сборка

```bash
npm run build     # продакшн-сборка в dist/
npm run preview   # локальный предпросмотр сборки
```

Человекопонятные URL (`/catalog/...`, `/product/...`, `/gift`) работают в dev и preview из коробки.

## Ветки и публикация

- **`main`** — исходный код для разработки (эта ветка).
- **`deploy`** — готовая статическая сборка, которую отдаёт GitHub Pages (демо для заказчика, без локальной сборки).

Обновить опубликованное демо одной командой:

```bash
npm run deploy
```

## Структура

```
src/
  assets/        изображения, шрифты, иконки
  scss/          стили: токены и абстракции, базовые, компоненты, секции, страницы
  js/
    api/         мок-слой данных
    modules/     поведение страниц и виджетов (каталог, товар, корзина, ...)
    components/  переиспользуемые рендер-функции (карточка товара, CTA, ...)
    utils/       вспомогательные функции
  partials/      Handlebars-партиалы (шапка, футер, секции)
public/          статика как есть (в т.ч. 404.html — SPA-фолбэк для GitHub Pages)
index.html       главная
catalog.html     каталог
product.html     страница товара
gift.html        подарочный сертификат
payment.html     оплата
delivery.html    доставка и возврат
privacy.html     политика конфиденциальности
contacts.html    контакты
account.html     личный кабинет
not-found.html   страница 404
```
